üì≤ Task_Manager_Whatsapp ‚Äì High-level Overview
The Task_Manager_Whatsapp.json file contains the full WhatsApp workflow: webhook, parsing, AI intent detection, commands, and responses. This section documents the behavior at a high level so the file can stay the single source of truth for detailed node configuration.‚Äã

Goals
Receive incoming messages from WhatsApp via Evolution API.

Decide whether the message is:

A command (/task ..., #to-do-list, etc.).

A natural language request about tasks.

Call the TaskFlow API (/api/tasks) and, when relevant, call Description_Enrich for AI enrichment.

Reply to the user with formatted WhatsApp messages (task lists, confirmations, errors).‚Äã

Typical flow
Webhook (Evolution API ‚Üí N8N)

Receives message payload (message, from, senderName, etc.).‚Äã

User resolution

Extracts phone number.

Resolves or creates a userId associated with that phone in TaskFlow.‚Äã

Intent detection

If the message starts with /task, it is treated as a structured command.

Otherwise, AI is used to interpret the user request (create/list/complete/remove tasks).‚Äã

Task API calls

POST /api/tasks to create tasks.

GET /api/tasks?userId=... to list tasks.

PATCH /api/tasks/:id to complete or edit tasks.

DELETE /api/tasks/:id to remove tasks.‚Äã

Optional enrichment

After creating a task, the workflow may call Description_Enrich to generate a richer description and checklist.‚Äã

Response formatting and send

Builds human-friendly messages with emojis, numbering, and shortened IDs.

Sends the final message back via Evolution API HTTP request to WhatsApp.‚Äã
